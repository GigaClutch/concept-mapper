## Architecture decisions (do not silently revisit)
- **D1 â€” Demo-first:** bounded corpus (Ethics domain, ~20 SEP articles). JSON file storage. No Neo4j, no backend API, no frontend framework until the demo proves valuable. Schema stays migration-ready.
- **D2 â€” Backbone-first, evidence-second:** the graph is generated from a curated registry using LLM knowledge (backbone), then grounded with SEP evidence quotes. Not mined from scraped text first.
- **D3 â€” Closed world (amended by D7):** batch pipeline stages (backbone, grounding) stay closed-world â€” unknown mentions go to `quarantine/proposed_nodes.json`, never straight into the graph. The D7 browse-time researcher is the one sanctioned exception: it may add `status: "provisional"` rows to the registry and graph live, but every such row is verbatim-grounded, dashed in the viewer, surfaced in `review.html`, and removed if rejected. Registry rows carry `wikidata_qid` where available (provisional rows have `qid_source: "unresolved"` for the next Wikidata pass).
- **D4 â€” Edges:** typed only, no RELATED_TO. Families: hierarchical (IS_A, PART_OF, SUBCATEGORY_OF), developmental (DEVELOPED_BY, EXTENDED_BY, DERIVED_FROM, INFLUENCED_BY), oppositional (CONTRASTS_WITH, CRITIQUES, RESPONDS_TO), bibliographic (AUTHORED_BY, INTRODUCED_IN). CONTRASTS_WITH is symmetric (`meta.symmetric_types`; viewer renders no arrowhead). Weight = `1 âˆ’ 0.5^n`, n = evidence items (backbone assertion = 1, +1 per distinct grounding article). Never store LLM confidence numbers. Every generated edge carries `extractor: {model, prompt_version, date}`.
- **D5 â€” Viewer:** ego-network view is primary (â‰¤150 nodes ever shown); atlas mode secondary. Static HTML, no server.
- **D6 â€” Evaluation:** gold set = canonical seed (~30 hand-written edges) + adversarial set (~20 traps: direction errors, polysemy, plausible-but-false) + verification sample (randomly drawn from real output, human-verified). Recall proxy: compare per-article extraction against SEP "Related Entries". Score every prompt/pipeline change against the gold set before adopting it.
- **D7 â€” Lazy, browse-time expansion:** the "ultimate build" behaviour â€” the graph researches itself where attention lands. When the viewer (served by `serve.py`) shows a node whose `research.status` is unset, an optional auto-research toggle (or a Research-now button) calls `research.py`: it finds the node's SEP entry, runs one `claude-haiku-4-5` pass (`rs-v1`, structured outputs) for verbatim-quoted edges + provisional new nodes/edges, applies them with provenance (`origin: "research"`, node `status: "provisional"`), recomputes weights + metrics, and stamps the node researched so it is never re-researched. Safety rails are non-negotiable: every quote is verbatim-checked; edge types obey D4 domain/range; a per-session budget cap bounds spend; and `serve.py` snapshots â†’ runs `validate.py` â†’ **rolls back on any failure**, so a bad expansion never lands. Provisional material is governed by D6's review loop, not trusted blind. Static no-server viewing is unaffected.

## Working practices
- Git from the start; commit per phase milestone. Code lives here, NOT in Google Drive (sync corrupts files mid-write â€” it has happened).
- Windows machine; venv: `.\venv\Scripts\Activate.ps1`.
- Every pipeline stage idempotent and re-runnable; cache scrapes, never re-hit SEP unnecessarily. Be polite: rate-limit, identify the client, respect robots.txt.
- Evidence quotes stay short with attribution + URL (SEP/IEP licensing).
- API spend is metered: LLM pipeline stages cache every response to disk (reruns free), default to `claude-haiku-4-5`, and take a `--max-cost` hard stop. The key sits in gitignored `.env`; keep it out of OneDrive and out of git.
- `validate.py` runs on every graph build: edge endpoints exist in registry, types allowed, weights in range, no duplicate (source,target,type), no orphan nodes, evidence quotes verbatim in cached source.
- When extraction quality is in question, the gold set decides â€” not vibes.
